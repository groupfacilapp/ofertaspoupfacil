<objective>
Close two verification gaps that block AUTH-03 and AUTH-04:

1. crypto.ts is orphaned — no production code calls encrypt() before writing credential data. This plan introduces src/lib/credentials.ts, a thin service layer that wires encrypt/decrypt into the write/read paths for marketplace_connections.encrypted_credentials and channel_connections.encrypted_config. It also imports env.ts, forcing ENCRYPTION_KEY validation at module load time.

2. trigger.config.ts contains the literal placeholder string '<TRIGGER_PROJECT_REF>'. This plan replaces it with process.env.TRIGGER_PROJECT_ID and adds TRIGGER_PROJECT_ID to both .env.local and .env.example.

Purpose: Make AUTH-03 and AUTH-04 verifiable — credentials written through this service are encrypted; the Trigger.dev worker config no longer carries an unfilled placeholder.
Output: src/lib/credentials.ts (new), trigger.config.ts (updated), .env.local (updated), .env.example (updated).
