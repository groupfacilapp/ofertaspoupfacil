1. Create `supabase/migrations/20260318000000_automation_rules.sql`:

```sql
-- automation_rules: per-user, per-marketplace automation config
CREATE TABLE public.automation_rules (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
  marketplace TEXT NOT NULL CHECK (marketplace IN ('amazon', 'mercadolivre', 'shopee', 'aliexpress')),
  rule_type TEXT NOT NULL CHECK (rule_type IN ('fetch', 'dispatch')),
  is_active BOOLEAN DEFAULT false,
  interval_minutes INT NOT NULL DEFAULT 60,
  start_hour INT NOT NULL DEFAULT 8 CHECK (start_hour >= 0 AND start_hour <= 23),
  end_hour INT NOT NULL DEFAULT 22 CHECK (end_hour >= 0 AND end_hour <= 23),
  target_group_id UUID REFERENCES public.dispatch_groups(id) ON DELETE SET NULL,
  last_run_at TIMESTAMPTZ,
  products_found_today INT DEFAULT 0,
  products_found_reset_at DATE DEFAULT CURRENT_DATE,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE (user_id, marketplace, rule_type)
);

-- RLS
ALTER TABLE public.automation_rules ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can manage own automation_rules" ON public.automation_rules
  FOR ALL USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);

-- Service role bypass
CREATE POLICY "Service role full access on automation_rules" ON public.automation_rules
  FOR ALL USING (current_setting('role') = 'service_role');

-- Indexes
CREATE INDEX idx_automation_rules_user ON automation_rules (user_id);
CREATE INDEX idx_automation_rules_active ON automation_rules (is_active, rule_type) WHERE is_active = true;

-- Add dispatched tracking index to dispatch_logs for date-range queries
CREATE INDEX IF NOT EXISTS idx_dispatch_logs_date ON dispatch_logs (dispatched_date, user_id);
```

2. Create `src/lib/queries/products.ts` with these exported functions:
- `getProductQueue(userId, filters)` - queries `offers` table joined with `dispatch_logs` to determine pending/sent status. Filters: marketplace, status (pending/sent), search (title ILIKE). A product is "pending" if no dispatch_log entry exists for it today, "sent" if one does. Returns array of offer rows with computed `status` field. Orders by fetched_at DESC. Limit 100.
- `getProductStats(userId)` - returns { total, pending, sentToday, failed } by counting offers and cross-referencing dispatch_logs for today.
- `markProductsSent(userId, offerIds)` - no-op placeholder (dispatch_logs insertion happens in dispatch flow). This just validates IDs belong to user.
- `clearSentProducts(userId)` - deletes offers that have been dispatched (have a sent dispatch_log entry today) for given user.

Use `supabaseAdmin` from `@/lib/supabase/admin`. All functions take `userId: string` as first param.

For `getProductQueue`, use this approach:
```typescript
// Fetch offers
const offersQuery = supabaseAdmin.from('offers').select('*').eq('user_id', userId).order('fetched_at', { ascending: false }).limit(100);
// Apply marketplace filter if provided
// Apply search filter with .ilike('title', `%${search}%`)
// Then fetch today's dispatch_logs for this user to build a Set of dispatched offer IDs

```
