1. Create `src/app/(dashboard)/produtos/actions.ts` with `'use server'` directive:

**`sendProduct(offerId: string)`** server action:
- Get current user via `createClient()` from `@/lib/supabase/server`
- Validate user owns the offer (query offers table with user_id + offer id)
- Find all active dispatch_groups for user that include this offer's marketplace
- For each group, get destinations from group_destinations
- Build a NormalizedOffer from the DB row
- Call dispatch logic: insert a dispatch_log entry, send via Evolution API (reuse the sending pattern from dispatch.ts -- import getEvolutionClient and makeInstanceName from platform-settings, formatMessage from dispatch)
- Return `{ success: boolean, error?: string, dispatched?: number }`
- Use `revalidatePath('/produtos')` after successful send

**`clearSentProducts()`** server action:
- Get current user
- Query offers that have at least one dispatch_log with status in (sent, delivered, read) today
- Delete those offers from the offers table (CASCADE will clean up dispatch_logs)
- Return `{ deleted: number }`
- Use `revalidatePath('/produtos')` after

2. Create `src/app/(dashboard)/produtos/page.tsx` (server component):
- Auth check via createClient() + getUser()
- Fetch products using supabaseAdmin from offers table (all user's offers, ordered by fetched_at DESC, limit 200)
- Fetch today's dispatch_logs for user to build dispatched set
- Fetch dispatch_groups for user (for the send action to know targets)
- Compute stats: total, pending (no dispatch_log today), sent (has dispatch_log today)
- Pass data to ProdutosClient component

Follow the same pattern as `src/app/(dashboard)/grupos/page.tsx` for data fetching.
