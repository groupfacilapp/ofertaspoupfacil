1. Create `src/app/(dashboard)/automacoes/actions.ts` with `'use server'`:

**`saveAutomationRule(formData)`** server action:
- Extract: marketplace, rule_type, is_active, interval_minutes, start_hour, end_hour, target_group_id
- Get user via createClient()
- Upsert into automation_rules: `supabaseAdmin.from('automation_rules').upsert({ user_id, marketplace, rule_type, is_active, interval_minutes, start_hour, end_hour, target_group_id, updated_at: new Date().toISOString() }, { onConflict: 'user_id,marketplace,rule_type' })`
- Return `{ success: true }` or `{ success: false, error: string }`
- Call `revalidatePath('/automacoes')`

**`toggleRule(ruleId: string, isActive: boolean)`** server action:
- Get user, verify rule belongs to user
- Update automation_rules set is_active = isActive where id = ruleId and user_id = user.id
- Return { success: boolean }
- Call `revalidatePath('/automacoes')`

2. Create `src/app/(dashboard)/automacoes/page.tsx` (server component):
- Auth check
- Fetch automation_rules for user (all rules, joined with dispatch_groups name via target_group_id)
- Fetch active dispatch_groups for user (for the group selector dropdown)
- Fetch stats: pending products count (offers without dispatch_log today), products found today (sum of products_found_today from fetch rules), last dispatch timestamp per marketplace (max last_run_at from dispatch rules)
- Fetch marketplace_connections to know which marketplaces are connected
- Pass all to AutomacoesClient
